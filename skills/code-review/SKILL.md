# コードレビュースキル

## 概要

指定されたコードまたは変更に対して、包括的なコードレビューを実施します。

## レビュー観点

$ARGUMENTS に対して、以下の観点でレビューしてください：

### 1. コード品質
- 可読性：変数名、関数名は適切か
- 構造：責務分離は適切か
- 重複：DRY原則に従っているか

### 2. セキュリティ
- 入力値の検証
- 認証・認可の適切な実装
- 機密情報の取り扱い

### 3. パフォーマンス
- 不要なループや計算
- メモリ使用量
- データベースクエリの効率

### 4. テスタビリティ
- テストしやすい設計か
- 依存関係の注入は適切か

## 出力形式

```markdown
## レビュー結果

### 良い点
- ...

### 改善提案
| 優先度 | 箇所 | 内容 |
|--------|------|------|
| 高     | ファイル:行番号 | 説明 |

### セキュリティ懸念
- ...

### 総評
...
```
